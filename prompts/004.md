以下のWebアプリケーションを構築しようと思います。
ソースコードを生成してください。

[アプリケーションの概要]

- タイトル: ドライブルート提案アプリ
- 機能:
    - ユーザーが移動手段を選択できるフォーム: 車、バイク、自転車、電車、バス
    - ユーザーが出発地を入力できるフォーム
        - 現在地を自動取得して、出発地とする機能
    - 目的地の候補を提案・表示する機能
        - 目的地1の候補として、出発地からの距離が100kmから300kmの範囲内にある地点からランダムに3つを提示するセレクトメニュー
        - 目的地2以降の候補として、目的地1からの距離が50kmから100kmの範囲内にある地点からランダムに3つを提示するセレクトメニュー
        - 提示された候補から、ユーザーが目的地を選択でき、選択された候補を目的地として設定し、次の目的地候補を提示する処理に移る
    - 地図を表示するUI（Leafletと地理院地図を利用）
    - 提案されたルートを地図上に表示
    - ルートの距離と推定所要時間を表示

[ロジック]

- 出発地をユーザーが入力するテキストボックスを表示する
- 出発地の入力を受け取る
- 目的地1の提案
    - 出発地からの距離が100kmから300kmの範囲内にある目的地1の候補を3つ抽出する
    - ページにセレクトメニューを動的に追加して、ユーザーが目的地1の候補を選択できるようにする
    - 目的地1の候補から1つをユーザーに選択させ、目的地1とする
    - ルート検索APIを利用して、出発地から目的地1への最適なルートを取得
    - 取得したルート情報を地図上に描画
- 目的地2の提案
    - 目的地1からの距離が50kmから100kmの範囲内にある目的地2の候補を3つ抽出する
    - ページにセレクトメニューを動的に追加して、ユーザーが目的地2の候補を選択できるようにする
    - 目的地2の候補から1つをユーザーに選択させ、目的地2とする
    - ルート検索APIを利用して、目的地1から目的地2への最適なルートを取得
    - 取得したルート情報を地図上に描画
- 目的地3の提案
    - 目的地2からの距離が50kmから100kmの範囲内にある目的地3の候補を3つ抽出する
    - ページにセレクトメニューを動的に追加して、ユーザーが目的地3の候補を選択できるようにする
    - 目的地3の候補から1つをユーザーに選択させ、目的地3とする
    - ルート検索APIを利用して、目的地2から目的地3への最適なルートを取得
    - 取得したルート情報を地図上に描画
- ルート全体の距離と推定所要時間、各区間の距離と推定所要時間を計算し、表形式で表示

[使用技術]

- フロントエンド: HTML, CSS, JavaScript
- 地図ライブラリ: Leaflet
- 地図タイル: 地理院地図
- ルート検索: OSRM API
- 住所から座標への変換（ジオコーディング）: Nominatim

出発地から目的地までの距離や、目的地同士の距離を算出する場合には、[municipalities.json](https://raw.githubusercontent.com/yzkn/Gemini-JS-DriveRouteSuggestion/refs/heads/main/master/municipalities.json) を読み込んで、そこに記載されている緯度・経度と、ヒュベニの公式を利用してください。
